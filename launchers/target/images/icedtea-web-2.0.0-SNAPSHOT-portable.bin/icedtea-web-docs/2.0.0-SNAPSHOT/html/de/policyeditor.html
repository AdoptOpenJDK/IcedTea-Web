    • NAME

    policyeditor — Einstellungen der Sicherheitsrichtlinie für javaws und das Browser Plug-in anzeigen und bearbeiten

    ÜBERSICHT

    policyeditor
    policyeditor [-file] Richtliniendatei

    BESCHREIBUNG

    policyeditor ist eine Anwendung mit grafischer Benutzeroberfläche sowie ein wenig Kommandozeilenunterstützung. Die Anwendung ermöglicht die Anzeige und Bearbeitung von Sicherheitsrichtlinieneinstellungen für Applets, die durch die Implementierungen von javaws und des Browser Plug-ins in IcedTea-Web verwendet werden. Sie ist als eine vereinfachte und leichter zu nutzende Alternative zum JDK Policy Tool gedacht. Administratoren und fortgeschrittene Benutzer, die eine fein granulare Kontrolle über Richtliniendateien brauchen, sollten das policytool statt des PolicyEditor verwenden.

    Werden keine Parameter bei der Ausführung angegeben, so wird keine Datei geöffnet, und eine Aufforderung zur Angabe des Speicherortes erscheint. Wird ansonsten ein Dateipfad als Parameter an der Kommandozeile angegeben, so wird der Dateipfad geöffnet und als eine Richtliniendatei verarbeitet.

    Fehlende Ressource: manCommandLineOptions_de

  • -codebase url - Gibt die Codebasis-URL eines Applets an. Falls die angegebene Codebasis in der Richtliniendatei bereits existiert, wird diese im geöffneten Editor markiert. Falls die Codebasis neu ist, wird diese zur Richtliniendatei hinzugefügt und im geöffneten Editor markiert. An die Option „-codebase“ können auch mehrere, durch Leerzeichen getrennte URLs übergeben werden. In diesem Fall werden alle Codebasen zur Richtliniendatei hinzugefügt und die letzte im geöffneten Editor markiert. Ausschließlich die erste in der Kommandozeile vorkommende Option „-codebase“ wird verarbeitet.(Genau ein Parameter erwartet)
  • -defaultfile - Specifies that the default user-level policy file should be opened. This is the file which is normally used by IcedTea-Web to make decisions about custom policies and permissions for applets at runtime, unless configured otherwise.(Kein Parameter erwartet)
  • -file policy_file - Gibt den Pfad der zu öffnenden Richtliniendatei an. Falls genau ein Parameter übergeben wird, der nicht diese Option ist, wird er als ein zu öffnender Dateipfad interpretiert, so als ob diese Option als erste angegeben werden wäre. Diese Option ist hauptsächlich aus Kompatibilitätsgründen zum Richtlinieneditor policytool verfügbar, wird aber auch zum Öffnen einer Richtliniendatei sowie mit der Option „-codebase“ verwendet.(Genau ein Parameter erwartet)
  • -help - Gibt Informationen über unterstützte Befehle und grundlegende Verwendung aus.(Kein Parameter erwartet)
  • -principals class_name principal_name - Specifies class name/principal name pairs (space-separated) for the policy entry identifier. This can be used with the other selector options to select a policy entry upon opening the editor; if no such identifier exists then it will be created and then selected.(Eine gerade Anzahl von Parametern mit „Parameter=Wert“ als gültigen Parameter erwartet)
  • -signedby certificate_alias - Specifies a certificate alias for a certificate stored in the keystore. This can be used with the other selector options to select a policy entry upon opening the editor; if no such identifier exists then it will be created and then selected.(Genau ein Parameter erwartet)
  • -verbose - Ausführliche Ausgabe aktivieren.(Kein Parameter erwartet)

    • BEISPIELE

  • policyeditor - Zeigt den GUI-Editor ohne eine geöffnete Datei an.
  • policyeditor -file %XDG_CONFIG_HOME\icedtea-web\security\java.policy - Zeigt den GUI-Editor mit der geöffneten Standardrichtliniendatei an.

    • DATEIEN

  • %XDG_CACHE_HOME\icedtea-web\cache\recently_used.cache - Additional information about items in cache
  • %XDG_CONFIG_HOME\icedtea-web\security\java.policy - Contains granted permissions for selected unsigned apps. Gesteuert durch deployment.user.security.policy

    • Where $XDG_CONFIG_DIR, $XDG_CACHE_DIR and $XDG_RUNTIME_DIR are set as ~/.config, ~/.cache and /tmp or /var/tmp if not set.
